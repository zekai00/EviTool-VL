# 本地 Qwen 当前模型训练下一步计划

- 生成时间：2026-05-27 22:23 CST
- 工作区：`/root/Workspace/VLM`
- 仓库：`/root/Workspace/VLM/EviTool-VL`
- 目标模型：本地 Qwen2.5-VL / Qwen3-VL，而不是 DashScope 远程模型。

## 1. 先把角色分清

`Teacher policy` 指教师策略：远程 `qwen3.6-flash-2026-04-16`、`qwen3.7-max` 这类模型可以帮我们验证环境、生成少量示范轨迹、做 baseline 对照，但它们的参数不在我们机器上，不能被我们的 RL 更新。

`Current model` 指当前模型：本项目真正要训练和更新的本地 Qwen2.5-VL/Qwen3-VL checkpoint。只有 current model 自己在环境里 rollout，再用 verifier reward 更新它的参数，才是严格的 `On-policy RL`（在线策略强化学习）。

所以当前正确链路是：

```text
DashScope teacher/baseline
  -> 验证环境可跑、动作 JSON 可执行、生成少量高质量 warmup

本地 Qwen2.5-VL / Qwen3-VL current model
  -> action JSON SFT warmup
  -> current model 自己 rollout
  -> verifier reward
  -> GRPO/RL 更新本地 checkpoint
```

## 2. 现在应该先做什么

优先级最高的是把“本地 current model 能输出合法 action JSON”做出来，而不是继续扩大远程 teacher rollout。

推荐顺序：

1. 做 rollout schema validator。
   - `Schema validator` 指轨迹格式检查器，用程序检查每条 rollout 是否有截图、任务目标、动作、reward、终止状态、policy_info 和错误信息。
   - 原因：后续 SFT/RL 都吃这些轨迹，格式不稳定会直接污染训练。

2. 构建本地 Qwen 的 action JSON SFT warmup 数据。
   - 数据来源一：本地 Playwright smoke tasks 的 scripted oracle 成功轨迹。
   - 数据来源二：DashScope teacher 成功轨迹，但必须由本地 verifier 确认 success=true。
   - 数据来源三：BrowserGym/MiniWoB++ 成功轨迹，先只收成功任务。
   - 目标不是大，而是让本地模型先学会稳定输出合法 JSON。

3. 训练本地 Qwen2.5-VL 或 Qwen3-VL 的 LoRA SFT。
   - `SFT` 指监督微调，让模型模仿正确 action JSON。
   - 建议先选小模型跑通，比如 Qwen2.5-VL-3B-Instruct 或本地可用的 Qwen3-VL 小尺寸模型。
   - 第一版只做 100-500 条高质量单步 action 样本，不追求规模。

4. 实现 local Qwen policy wrapper。
   - `Local policy wrapper` 指本地模型策略封装器，把本地 checkpoint 的输出接入同一个 rollout collector。
   - 它必须替代 `qwen_dashscope`，成为真正的 current model policy。

5. 用本地 current model 跑 rollout baseline。
   - 环境：先本地 Playwright smoke tasks，再 BrowserGym/MiniWoB++。
   - 指标：success rate、valid JSON rate、valid action rate、avg steps、reward。
   - 这一步开始才接近真正 on-policy RL。

6. 启动 Verifier-Guided GRPO。
   - `Verifier-Guided GRPO` 指由 verifier reward 引导的 GRPO 强化学习。
   - 每轮用当前本地模型采样多条轨迹，按 reward 更新模型。
   - 更新后产生的新 checkpoint 才是下一轮 current model。

## 3. 为什么不是马上 RL

马上 RL 风险很高：

1. 本地 Qwen 还不一定能稳定输出合法 JSON。
2. 如果 action parse 失败率高，RL reward 会极稀疏，训练会浪费大量算力。
3. 如果 rollout schema 没检查，坏轨迹会混进训练。
4. 远程 teacher 成功不等于本地模型成功，不能直接跳过本地 SFT warmup。

因此合理路径是：

```text
schema validator
-> 高质量 action JSON SFT warmup
-> 本地模型 rollout baseline
-> 小规模 GRPO
-> 扩任务和扩环境
```

2. 汇总 SFT warmup 数据。

3. 选择本地 base model。
   - 优先检查 `/root/models` 里已有哪个 Qwen2.5-VL/Qwen3-VL。
   - 先用能稳定加载的最小模型跑 LoRA SFT smoke。

4. 跑 SFT dry-run。
   - 目标：证明数据格式、图片路径、tokenizer/template、LoRA 配置都能跑。
   - 不追求最终效果。

5. 接本地模型 policy wrapper。
   - 目标：让训练后的本地 checkpoint 在 `run_browser_rl_rollouts.py` 里作为 `--policy local_qwen` 使用。
