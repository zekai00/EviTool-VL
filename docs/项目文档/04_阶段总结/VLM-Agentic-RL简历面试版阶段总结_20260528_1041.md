## 项目一句话

构建了一个面向视觉语言模型的 GUI agentic RL 训练与评测框架，使 Qwen2.5-VL 能够看浏览器截图、输出结构化 GUI 动作，并在可执行环境中完成多步网页任务；通过 1000 个任务、轨迹增强和 history-aware SFT，将独立 test100 成功率提升到 0.82。

## 面试讲述版本

这个项目的目标是让视觉语言模型不只是回答图片问题，而是能像 GUI agent 一样看截图、理解目标、连续执行操作。最开始我尝试过候选框选择式 GUI RL，也就是先生成一批可点击候选框，再让模型从里面选择。但这条路线的上限很受候选池召回影响，容易变成选框排序任务，不是真正的多步 agent。

后来我把主线调整为可执行环境中的 VLM agentic RL：用 Playwright 搭了一个本地浏览器环境，定义统一的 action JSON，包括 click、type、press、scroll、finish；每个任务都有 reset、step、screenshot、verifier 和 reward。这样模型每一步都能看到真实截图，输出动作，然后环境执行并返回结果。

数据方面，我构建了 1000 个本地 browser GUI-RL 任务，覆盖表单、搜索、菜单、表格、待办、选择控件和组合任务。为了让模型不只记住中心点坐标，我做了坐标扰动增强；为了让模型学会失败后恢复，我加入了失败恢复示范。最终生成 2600 条 oracle rollout，并转成 7942 条 history-aware trajectory SFT 样本。history-aware 的意思是每一步提示里都有历史动作、上一步 verifier progress 和当前 step，模型不是盲目输出下一步，而是根据状态做决策。

训练上，我用本地 Qwen2.5-VL-3B-Instruct 做 base model，用 LoRA 和 4bit 量化训练 1 个 epoch。评测是在独立 test100 可执行任务上做的，不是离线 accuracy。最终 success_rate 达到 0.82，valid_json_rate 和 valid_action_rate 都是 1.0。相比 300 任务 v2 的 0.333，有明显提升。

目前这个结果还不是完整 on-policy RL，而是进入 on-policy RL 前的 SFT warmup 和环境闭环。下一步我会用当前训练出的 LoRA 作为 current model，让模型自己 rollout，再用 verifier reward 做小规模 GRPO 或 verifier-guided RL，让模型从自己的失败轨迹里学习。

## 面试高频问题回答

### 1. 这个项目和普通 VLM SFT 有什么不同？

普通 VLM SFT 多数是图像到文本答案，而这个项目训练的是状态到动作。输入包含截图、任务目标、历史动作和 verifier progress，输出是可执行 GUI action JSON。评测也不是文本匹配，而是真实浏览器环境中的任务成功率。

### 2. 为什么不继续做候选框选择？

候选框路线容易被候选池召回限制。如果正确目标没有被生成，模型无法选中。更重要的是，它会把问题变成候选框排序，而我的目标是让 VLM 学会多步 GUI 操作。因此候选框可以作为辅助，但主线应是 direct GUI action 和可执行环境。

### 3. 为什么先做 SFT，而不是直接 RL？

直接 RL 的搜索空间太大，模型一开始可能连合法 JSON action 都输出不稳定。SFT warmup 先让模型学会动作格式、坐标规则和基础任务流程。当前 valid_json_rate 和 valid_action_rate 已经到 1.0，这说明它已经适合作为 current model 进入 on-policy RL。

### 4. 你怎么保证数据质量？

任务有程序化 verifier，oracle rollout 会真实执行；SFT 数据按 task split 切分，避免 train/test 泄露；rollout 经过结构校验、动作解析校验、执行状态校验和截图字段校验。当前 2700 条 rollout 校验无错误。

### 5. 当前最大不足是什么？

当前还没有完成严格 on-policy RL，只完成了可执行环境、SFT warmup 和评测闭环。任务也主要是本地 browser synthetic tasks，不是 OSWorld 级复杂系统任务。另外 form 和 choice 仍是主要弱项，需要通过定向数据和 verifier-guided RL 改进。

### 6. 下一步怎么做？

下一步用 v3 LoRA 作为 current model，开始小规模 verifier-guided GRPO。每个任务采样多条 rollout，用 verifier reward 计算优势，强化成功动作，惩罚无效动作、重复点击和无进展状态。算法闭环稳定后，再迁移到 Qwen2.5-VL-7B。
