## 面试讲述版本

这个项目的目标是让视觉语言模型不只是回答图片问题，而是像 GUI agent 一样看截图、理解目标、连续执行操作。早期我尝试过候选框选择式 GUI RL，也就是先生成一批可点击候选框，再让模型从里面选目标。但这条路线受候选池召回限制，容易变成选框排序任务，不是真正的多步 agent。

后来我把主线调整为可执行环境中的 VLM agentic RL。具体来说，我用 Playwright 搭了一个本地浏览器环境，统一 action JSON，包括 click、type、press、scroll、finish；每个任务都有 reset、step、screenshot、verifier 和 reward。这样模型每一步都能看到真实截图，输出动作，然后环境执行并返回结果。

数据上，我先构建 1000 个任务做 history-aware trajectory SFT。history-aware 的意思是每一步输入里包含当前截图、任务目标、历史动作、上一步 verifier progress 和动作空间，让模型学习状态到动作，而不是单张图到答案。通过坐标扰动和失败恢复示范，Qwen2.5-VL-3B 在独立 test100 上达到 0.82 success_rate，JSON/action 合法率都是 1.0。

随后我构建了 2000 任务正式套件，train/val/test=1600/200/200，并把 v3 SFT 模型在 val200 上的 0.745 作为正式 baseline。接着实现 verifier-guided on-policy GRPO：当前模型自己采样动作，环境执行，verifier 给 reward；同一状态下多个动作组成 group，用组内相对 reward 更新 LoRA adapter。第一轮 100 trainable groups 的安全版训练后，val200 提升到 0.790。

当前模型是 `/root/models/Qwen2.5-VL-3B-Instruct` 加上 `outputs/onpolicy_browser_rl_grpo_100tg_safe_20260528_1646/adapter`。这个 adapter 是 LoRA adapter；它是从 v3 SFT LoRA 继续训练后保存的完整 adapter，推理时不需要再叠两层 adapter。

## 面试高频问题回答

### 1. 这个项目和普通 VLM SFT 有什么不同？

普通 VLM SFT 多数是图像到文本答案；这个项目训练的是状态到动作。输入包含截图、任务目标、历史动作和 verifier progress，输出是可执行 GUI action JSON。评测也不是文本匹配，而是真实浏览器环境中的任务成功率。

### 2. 为什么不继续做候选框选择？

候选框路线受候选池召回限制。如果正确目标没有被生成，模型无法选中。更重要的是，它会把问题变成候选框排序，而目标是让 VLM 学会多步 GUI 操作。因此候选框可以作为辅助，但主线应是 direct GUI action 和可执行环境。

### 3. 为什么先做 SFT，再做 RL？

直接 RL 的搜索空间太大，模型一开始可能连合法 JSON action 都输出不稳定。SFT warmup 先让模型学会动作格式、坐标规则和基础流程。之后再做 on-policy GRPO，让模型从自己的成功和失败中学习。

### 4. 现在是否已经是真正 on-policy RL？

是，已经完成了第一轮真正 on-policy RL 闭环。当前模型自己采样动作，环境执行动作，verifier 给 reward，GRPO 用这些当前策略产生的数据更新 LoRA。第一轮规模较小，是 100 trainable groups，val200 从 0.745 提升到 0.790。

### 5. 当前模型是什么组合？

当前模型是 Qwen2.5-VL-3B-Instruct base model 加一个 LoRA adapter：

```text
/root/models/Qwen2.5-VL-3B-Instruct
+ outputs/onpolicy_browser_rl_grpo_100tg_safe_20260528_1646/adapter
```

这个 adapter 已经包含从 v3 SFT 继续训练后的权重变化，推理时不需要再叠 v3 adapter。

### 6. 当前最大不足是什么？

还没跑 test200，所以不是最终模型。advanced 和 table 在 val200 上有退化，说明 RL 还需要 replay 或 KL 稳定项。任务也仍是本地 synthetic browser tasks，还没有扩展到复杂真实网页或 OSWorld。

### 7. 下一步怎么做？

下一步扩大到 200-300 trainable groups，加入 v3 replay 或 KL 稳定项，重点补 advanced_scroll/dialog、choice_checkbox、table_action。若 val200 达到 0.82 以上且主要 family 不退化，再跑 test200 做阶段性验收。3B 闭环稳定后，再迁移到 Qwen2.5-VL-7B。
