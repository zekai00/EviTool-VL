## 当前限制

1. 1000 unique source 已完成，但最终严格 trajectory 只有 559 条。

原因不是 source 不够，而是候选召回仍然是上限瓶颈。严格版要求候选框本身达到 IoU@0.5，所以只有通过候选质量门槛的样本能进入主数据。

2. DashScope VLM 补框本轮不可用。

`gui-plus` 免费额度已耗尽；用户列出的 `qwen3.7-max` 系列在当前图片消息格式下不可用。后续若要继续 VLM 补候选，需要找到真正支持 image input 的可用模型，或调整 DashScope 的多模态调用格式。

3. enhanced visual/layout 融合脚本在 1000 unique image 上太慢。

瓶颈出现在大量候选的 O(n^2) 去重逻辑。后续应把 `_dedupe_candidates` 改成网格/hash 近邻去重，或者先按 source 限流再去重。

4. label / page-tab / tree-item 仍偏弱。

这些类别在没有 OCR/VLM 的情况下仍难做，尤其 label 的 IoU@0.5 很低。

## 建议

当前默认使用严格版做下一轮 v2 SFT：

- 规模比之前 77 张 unique train image 明显改善到 442 张。
- 格式更干净：local shuffled candidate id、无 rank/score、select_candidate 而不是 bbox copy。
- 质量闭合：最终 IoU@0.5 为 100%。

宽松版可以作为后续 ablation 或低权重混入，但不建议先用它做主训练。

下一步优先级：

1. 用严格版 v2 做一次 SFT，重新做 teacher-forced select_candidate eval。
2. 优化 enhanced candidate 去重性能，重跑 1000 unique 本地增强。
3. 找到可用的 DashScope 图像模型后，只对低召回类别做 VLM 补候选。
4. 再构建一版 1000 unique high-recall full-aware/v2 数据。
