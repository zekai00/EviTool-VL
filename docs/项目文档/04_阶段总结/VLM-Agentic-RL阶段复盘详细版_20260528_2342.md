## 关键技术名词

- `VLM`：Vision-Language Model，视觉语言模型。这里指能同时理解图像和文本的模型，例如 Qwen2.5-VL、Qwen3-VL。
- `GUI`：Graphical User Interface，图形用户界面。网页、桌面软件、表单、菜单、表格都属于 GUI。
- `Agentic RL`：智能体强化学习。模型不是只回答静态问题，而是在环境中连续观察、行动、获得奖励并学习策略。
- `BrowserRL`：浏览器强化学习环境。这里指我们用本地 HTML 任务、Playwright 和 verifier 组成的可 reset、step、reward 的 GUI 环境。
- `Playwright`：浏览器自动化工具。它能用程序打开网页、点击、输入、截图；我们用它执行模型输出的 GUI action。
- `Verifier`：验证器。它用程序检查页面状态，例如表单是否填对、目标项是否选中，并返回 success 和 reward。
- `Rollout`：一次完整任务尝试。从 reset 开始，模型多步执行动作，直到成功、失败或达到最大步数。
- `History-aware trajectory SFT`：带历史信息的轨迹监督微调。输入包含截图、目标、历史动作、verifier progress，输出下一步 action。
- `LoRA adapter`：低秩适配增量权重。base model 不全量训练，只训练少量 LoRA 参数；adapter 是泛称，LoRA 是具体类型。
- `On-policy RL`：在线策略强化学习。训练数据来自当前模型自己在环境中采样的动作，而不是只用 scripted oracle 专家轨迹。
- `GRPO`：Group Relative Policy Optimization。对同一状态采多个动作，用组内 reward 相对高低计算优势，提升高 reward 动作概率。
- `Trainable group`：可训练动作组。同一状态下多次采样动作，如果 reward 有差异，就能提供 RL 学习信号。
- `Replay`：重放旧的高质量数据或旧策略采集数据，防止新训练把已有能力遗忘。
- `Stream collect`：流式采集。每采到一个 group/rollout 就写入临时 JSONL，避免中断后丢失数据。
- `Resume`：恢复采集。从已有临时文件读取已采数据，跳过已完成 task，继续采未完成部分。
- `Val gate`：验证集门槛。用固定验证集判断 adapter 是否值得进入下一阶段。
- `Test set`：测试集。只用于阶段性最终验收，不应频繁调参使用。

## 当前模型能做什么

当前模型已经不是候选框选择模型，而是 direct GUI action 模型。它能：

1. 读取浏览器截图和文本目标。
2. 根据历史动作和 verifier progress 判断下一步。
3. 输出合法 JSON action，例如 `click`、`type`、`press`、`scroll`、`finish`。
4. 使用 0-1000 归一化坐标点击页面元素。
5. 在 Playwright 浏览器环境中真实执行多步任务。
6. 完成本地 BrowserRL 中的 form、search、menu、table、todo、choice、advanced 等任务。

它当前还不能称为通用桌面 agent，也不能保证泛化到任意真实网站或 OSWorld。

## 阶段 1：路线从候选框转向可执行 Agent

早期我们做过 GUI candidate 路线：先生成候选框，再让模型从候选框中选择目标区域。这条路线工程上可控，但上限受候选池召回限制。如果目标框不在候选池里，模型无法选中；更重要的是，它容易把任务变成“选框排序”，而不是让 VLM 学会多步 GUI 操作。

因此当前主线已经明确为：

```text
截图 + 目标 + 历史状态 -> 模型输出 action JSON -> 环境执行 -> verifier reward -> on-policy RL 更新
```

候选框、full-candidate score cache、candidate-only GRPO 只作为辅助方向，不是当前第一优先级。

## 阶段 2：BrowserRL 环境与任务体系

我们没有一开始就上 OSWorld，而是先用 Playwright 搭建本地浏览器环境。这样 reset、step、verifier 都更可控，适合快速验证算法闭环。

当前环境支持：

- 本地 HTML 任务页面。
- 每一步截图保存。
- 统一 JSON action schema。
- 0-1000 归一化坐标映射到真实浏览器坐标。
- verifier 自动判分。
- rollout 记录 task id、goal、action、reward、screenshot、execution status。

## 阶段 7：采集器流式落盘与 Resume

213tg 实验暴露了 collect-only 的工程问题：长采集过程中只有结束后才写 `groups.jsonl`，中断后可能丢失 partial groups，也无法实时看到 trainable group 数量。

已修复：

- 默认开启 `--stream-collect`
- 新增 `--stream-flush-every`
- 新增 `--resume-collect`
- 实时写：
  - `groups.jsonl.tmp`
  - `rollouts.jsonl.tmp`
  - `live_summary.json`
- 正常结束仍写：
  - `groups.jsonl`
  - `rollouts.jsonl`
  - `collect_summary.json`
  - `summary.json`
- resume 时读取临时文件，跳过已覆盖 task。
- 如果恢复后目标已满足，直接退出，不加载本地 Qwen。

验证目录：

```text
outputs/onpolicy_collect_stream_smoke_20260528_check
```

验证结果：

- 真实小采集：1 group / 1 rollout
- `groups.jsonl.tmp` 正常生成
- `rollouts.jsonl.tmp` 正常生成
- `live_summary.json` 正常更新
- resume 后没有重复采集，也没有重新加载模型
